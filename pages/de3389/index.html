celery后台运行

@[toc]

# celery后台运行

celery后台运行需要借助Supervisor进行管理程序

# 安装Supervisor

pip install supervisor
1

安装后 ,会多出echo_supervisord_conf、supervisorctl和supervisord 命令,后续有用

# Supervisor配置

进入到Django项目的uwsgi.ini同级目录,执行:

echo_supervisord_conf > supervisord.conf
1

运行后会多出一个supervisord.conf文件

使用vim编辑该文件,在最后添加:

[include]
files = /etc/superv.ini
1
2

再创建superv.ini文件,添加以下内容

superv.ini:

[program:celery.worker]
;指定运行目录
directory=/usr/factory_policy
;;运行目录下执行命令
command=celery -A factory_policy worker -l info
;
;;启动设置
numprocs=1          ;进程数
autostart=true      ;当supervisor启动时,程序将会自动启动
autorestart=true    ;自动重启
;
;;输出日志
stdout_logfile=/usr/factory_policy/factory_policy/celery_work.log
stdout_logfile_maxbytes=1MB ;默认最大1M
stdout_logfile_backups=10 ;日志文件备份数，默认为10
;;启动设置
numprocs=1      ;进程数
autorestart=true        ;是否自动重启
;;错误日志
redirect_stderr=true ;为true表示禁止监听错误

# 启动和关闭和重启Supervisor

在有supervisord.conf文件的路径下,执行

# 启动
supervisord -c supervisord.conf
# 关闭
supervisorctl -c supervisord.conf shutdown
# 重启
supervisorctl -c supervisord.conf reload
